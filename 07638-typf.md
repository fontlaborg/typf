# Extension Plan: Typf + Differentiable Variable Fonts

## Objective
Enable `typf` to act as a high-performance differentiable rendering provider for variable fonts, bridging the gap between Rust-based rendering and Python-based optimization.

## Implementation Phases

### 1. Differentiable Path Geometry (`typf-core`)
- Implement a `DifferentiablePath` type that tracks the derivation of control points from variation axes.
- Expose the Jacobian (partial derivatives) of each Bézier control point $(x, y)$ with respect to each axis $w_i$.
- Support the `support_function` logic in native Rust for speed.

### 2. Gradient-Enabled Backends
- **Vello Integration**: Extend the `typf-render-vello` backend to support rendering from `DifferentiablePath`.
- **JSON Metadata**: Add a `typf-render-json-diff` backend that exports both the interpolated geometry AND its gradients for use in external optimizers (like PyTorch).

### 3. Python Bindings (`typfpy`)
- Update the Python bindings to expose the `DifferentiableInterpolator` logic.
- Ensure that the `torch` tensors can be populated directly from the Rust-computed geometry and gradients using zero-copy mechanisms where possible.

### 4. GPU-Accelerated Support Functions
- Implement the variation support functions as compute shaders in the `vello` pipeline to allow real-time differentiable rendering of complex variable fonts.

## Success Criteria
- [ ] Render a variable font glyph and its gradients in <2ms.
- [ ] Successfully pass gradient checks (finite difference vs. analytical) for a 10-axis variable font.
